State Park Vegetation Map

This web map application was created for the final Practicum course of UW-Madison's GIS Development Master's program to allow users to find information on plant species which may be present in state parks using spatial information on natural communities. The project involed my independent proposal and creation of a web application showcasing holistic knowledge acquired throughout the program. The full report can be viewed here.
The application was designed with two major goals in mind: interface usability for the user, the general public, and independent, modular functions to allow for ease of data modification due to the complex and fluid nature of the source materials.
Databases were created in ArcPRO and hosted on ArcGIS Online, webpage functions used JavaScript and map functions used the ArcGIS Online API for JavaScript.

The Interface





A slippy map of Michigan with boundaries of all state parks is shown to the user. Clicking on polygon will highlight and populate the side panel with a list of species. Species are separated by their general structure (Tree, Shrub, Forb(flower) etc.)


Opening each panel will show information about the species and allow the user to search for parks containing that specific species.

Searching for parks highlights and returns park lists in the same manner as the species. Opening the individual blocks allows users to search inside the specific park.

How It Works

While the display and user functions are simple, there are complex functions occuring in the script. Selecting each park polygon then selects the spatially associated natural community areas which are not displayed on the map. These natural communities and their associated species come from the Michigan Natural Features Inventory .

Each of the natural communities has many species associated with it, and a species could have multiple natural communities, and so a many-to-many relationship is created between the two features. The script selects from each selected community type and outputs the table information to be placed in the side panel. This setup allows for a function to backsearch for parks which intersect natural communities to be saved in a button in the species description. This satisfies and showcases the goal of handling the data in a modular and independent manner, easily allowing for future components to be added without major modifications being required for the pre-existing code.
